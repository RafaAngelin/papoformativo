Sério, o que mais você deseja?

Você provavelmente constrói sites e acredita que seu projeto é especial. Você acha que sua página inicial com paralaxe de 13 megabytes vai ganhar algum prêmio que você pode exibir com orgulho no seu site. Você acredita que seu arquivo jQuery de 40 megabytes e 83 polyfills vai impressionar o Internet Explorer 7 porque finalmente apresenta sombra. Errado. Deixe-me descrever seu site ideal:

  • O site é leve e carrega rapidamente
  • Se adapta a todas as telas
  • Tem a mesma aparência em todos os navegadores
  • É acessível para todos que o visitam
  • É legível e transmite a sua mensagem (se você tiver uma, em vez de apenas fotos grandes de pessoas)

Bem, adivinha:

Você está exagerando. Olhe para este site. É apenas um site. Por que você precisa animar uma bandeira quando passa o mouse sobre ela? Você gastou horas nisso e adicionou 80 kilobytes ao seu site, e algumas pessoas que acessarem pelo iPad nunca verão isso. Sem mencionar que pessoas com deficiência visual nunca verão isso, mas elas não veriam de qualquer maneira.

Você não sabia disso, mas este é o site perfeito:

É leve

Esta página inteira pesa menos do que o logotipo gradiente do Facebook no seu site WordPress. Você realmente carregou 100kb do jQuery UI apenas para animar o fundo de uma div? Você carregou todas as 7 fontes de uma webfont apenas para dizer "Oi" no início do seu site? Isso é desnecessário.

É responsivo

Responsive significa que se adapta ao tamanho da tela em que é visualizado. Este site não se importa se você está em um iMac ou em um dispositivo pequeno.

Funciona bem

Este site é legível e tem uma hierarquia clara. Ele utiliza tags HTML5 para fornecer estrutura semântica.

Este é um site de verdade. Olhe para ele. É bonito.

Sim, isto é sátira

Não estou dizendo que seu site deve ser assim. Estou dizendo que muitos problemas com sites são criados por nós mesmos. Sites não são quebrados por padrão; eles são funcionais, de alto desempenho e acessíveis. Nós os tornamos complicados.
